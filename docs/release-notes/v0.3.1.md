# Obsidian Mneme 0.3.1 — First Public Release

Erster öffentlicher Release. **Obsidian Mneme** ist ein lokaler MCP-Server für semantische Obsidian-Vault-Suche — komplett lokal, keine Cloud, keine API-Keys.

*Der Name: **Mneme** (altgriechisch Μνήμη, "Erinnerung") war in der griechischen Mythologie eine der drei ursprünglichen Musen — Personifikation des Gedächtnisses. Claude fragt, Mneme erinnert sich.*

## Wofür ist das?

Obsidians eingebaute Volltextsuche findet nur exakte Strings. **Obsidian Mneme liefert semantische Suche** — findet Inhalte nach Bedeutung, auch wenn du andere Wörter verwendet hast. Drei Wege, das zu nutzen — wähl einen oder kombiniere:

### 1. Direkt in Obsidian (Plugin)

**Kein LLM nötig.** Ribbon-Icon klicken → Search-Sidebar → Query tippen. Hybrid-Search (Semantik + Keyword) auf deinem Vault. Läuft komplett offline. Zusätzlich: "Similar Notes"-Tab für die aktuelle Notiz und Vault-Health-Modal (Orphans / Stale / Weak-Links).

### 2. Mit Claude Desktop / Claude Code / Cursor / jedem MCP-Client

LLM-Session bekommt semantischen Zugriff auf deinen kompletten Vault via MCP. Beispiele:

- *"Was hab ich letzten Monat über Thema XY notiert?"* — findet die Notiz auch mit anderen Wörtern.
- *"Erinnere mich an den Architektur-Entscheid aus Projekt Z."* — zieht den relevanten Projektordner inkl. verwandter Notizen.
- *"Welche Notizen sind verwandt mit der hier?"* — semantische Nachbarn über Dense-Embeddings + Wikilink-Graph.

### 3. Von der CLI

`mneme search "luhmann zitat"` — Volltext + Semantik gleichzeitig. Auch `mneme similar <pfad>`, `mneme health`, `mneme vault-health`. Für Scripts, Automation, oder schnelles Nachschlagen ohne GUI.

**Dein Vault bleibt auf deinem Rechner.** Kein Upload, keine API-Keys, keine Telemetrie. BGE-M3 läuft lokal (CPU oder GPU).

## Für wen ist das?

- **Obsidian-Poweruser** mit wachsendem Vault (50+ Notizen) — du willst bessere Suche als Obsidians Standard-FTS
- **Privacy-bewusste** Menschen — alles lokal, keine Cloud
- **KI-Chat-Nutzer** (Claude Desktop / Claude Code / Cursor / Continue), die LLM-Kontext mit persönlichem Wissen anreichern wollen — *optional, nicht erforderlich*
- **Entwickler**, die einen lokalen RAG-Sidecar brauchen — kein LLM-Frontend

**Obsidian Mneme ist nicht:** ein Chat-UI, ein LLM-Frontend, ein Zettelkasten-Ersatz oder eine Cloud-SaaS-Alternative. Es ist ein **Retrieval-Sidecar** für deinen bestehenden Obsidian-Workflow.

## Features

- **Hybrid Search** — BGE-M3 Dense-Embeddings (1024-dim, multilingual) + BM25 fusioniert via Reciprocal Rank Fusion
- **MCP-native** — 8 Tools (`search_notes`, `get_similar`, `get_note_context`, `vault_health`, …), 3 Resources, 3 Prompts. Transport: stdio oder Streamable-HTTP
- **Obsidian-Plugin mit Auto-Start** — Server startet beim Öffnen von Obsidian automatisch. Search-Sidebar, Similar-Notes, Vault-Health-Modal, Status-Bar
- **Graph-aware Retrieval** (opt-in) — Wikilink-Graph für Centrality-Gewichtung (GARS)
- **CrossEncoder-Reranking** (opt-in) — zweite Pass für Präzisions-kritische Queries (ab ~500 Notizen sinnvoll)
- **Live-Watcher** — inkrementelles Reindexing bei File-Änderungen. Global-Coalescing-Debounce verhindert Lock-Contention bei Bulk-Saves (Obsidian-Sync, git-merge, rsync)
- **Vault Health (Gardener)** — findet Orphans, stale Notes, Duplikate, Weak Links
- **Golden-Dataset-Evaluator** — `mneme eval` misst Hit@k + MRR
- **GPU-aware** — dynamisches Backend (CPU / CUDA / ROCm). 80x+ Speedup auf AMD RX 7900 XTX mit ROCm + float16
- **Security-hardened** — Host-Header-Parsing gegen DNS-Rebind, Plugin-Spawn-Guard gegen RCE via Vault-synced `data.json`
- **296 Python-Tests + 26 Plugin-Tests**, CI auf Windows + Ubuntu für Python 3.11 / 3.12

## Installation

### Server

```bash
pip install obsidian-mneme
mneme setup            # Wizard: Vault-Pfad, Transport, Auto-Search
```

### Claude Desktop

Mneme in `claude_desktop_config.json` eintragen — [60-Sekunden-Quickstart im README](https://github.com/MakaveliGER/obsidian-mneme#60-sekunden-quick-start-claude-desktop).

### Obsidian-Plugin

Die drei Dateien unten (`main.js`, `manifest.json`, `styles.css`) herunterladen und nach `<your-vault>/.obsidian/plugins/mneme/` kopieren. In Obsidian: **Settings → Community Plugins → Mneme aktivieren**. Keine npm-Installation nötig. Details: [Plugin-README](https://github.com/MakaveliGER/obsidian-mneme/blob/main/obsidian-plugin/README.md).
