## Block 3: Markdown Parser

### Step 3.1 — Frontmatter + Wikilink + Tag Parsing

**Datei:** `src/mneme/parser.py`

- `parse_frontmatter(content: str) -> tuple[dict, str]` — YAML zwischen `---`, gibt (metadata, body) zurück
- `extract_wikilinks(text: str) -> list[str]` — Regex `\[\[([^\]|]+)(?:\|[^\]]+)?\]\]`, gibt nur Ziel zurück
- `extract_tags(frontmatter: dict, body: str) -> list[str]` — Tags aus `frontmatter["tags"]` + Inline `#tag` (Regex `(?:^|\s)#([a-zA-Z][\w/-]*)`)
- `ParsedNote` dataclass: `path, title, content_hash, frontmatter, tags, wikilinks, body`
- `parse_note(path: Path, vault_root: Path) -> ParsedNote`

### Step 3.2 — Heading-aware Chunking

**Datei:** `src/mneme/parser.py` (erweitern)

- `Chunk` dataclass: `content, heading_path, chunk_index`
- `chunk_note(parsed: ParsedNote, max_tokens: int, overlap_tokens: int) -> list[Chunk]`
  1. Split body an `^#{2,6}\s` Regex (Zeilenanfang + 2+ Hashes)
  2. Heading-Pfad akkumulieren basierend auf Heading-Level
  3. Semantic Context Header prependen: `[Title: X | Folder: Y | Tags: a, b]`
  4. Sections > max_tokens: Split an Doppel-Newlines
  5. Sections < 50 Tokens: mit nächster Section mergen
  6. Overlap: letzte overlap_tokens vom Vorgänger-Chunk prependen

Token-Zählung: Wort-basiert (split auf Whitespace), nicht Tokenizer — ausreichend genau für Chunking.

### Step 3.3 — Parser-Tests

**Datei:** `tests/test_parser.py`

- Test: Frontmatter mit Tags, ohne Tags, ohne Frontmatter
- Test: Wikilinks `[[Ziel]]`, `[[Ziel|Alias]]`, `[[Pfad/Ziel]]`, keine Links

## Block 5: Store

### Step 5.1 — SQLite Store mit sqlite-vec + FTS5

**Datei:** `src/mneme/store.py`

- `Store(db_path: Path, embedding_dim: int)`
- `__init__`: DB erstellen, sqlite-vec Extension laden, Schema anlegen (notes, chunks, chunks_fts, chunks_vec)
- FTS5-Sync via Trigger (INSERT/DELETE auf chunks → chunks_fts synchron halten)
- `upsert_note(path, title, content_hash, frontmatter, tags, wikilinks) -> int` — INSERT OR REPLACE, gibt note_id zurück
- `upsert_chunks(note_id, chunks: list[ChunkData])` — DELETE alte Chunks der note_id, INSERT neue, Vektoren in chunks_vec
- `delete_note(path)` — CASCADE löscht Chunks, Trigger löscht FTS5/vec Einträge
- `get_note_by_path(path) -> Note | None`
- `vector_search(query_embedding, top_k) -> list[SearchResult]` — KNN auf chunks_vec, JOIN chunks + notes
- `bm25_search(query_text, top_k, filters?) -> list[SearchResult]` — FTS5 MATCH, JOIN notes, WHERE-Filter
- `get_stats() -> IndexStats`
- Context Manager für Transaktionen

## Block 6: Search

### Step 6.1 — Hybrid Search + RRF

**Datei:** `src/mneme/search.py`

- `SearchEngine(store: Store, embedding_provider: EmbeddingProvider, config: SearchConfig)`
- `search(query: str, top_k: int, tags?, folders?, after?) -> list[SearchResult]`
  1. Query embedden
  2. `store.vector_search(embedding, top_k=top_k * 3)` — Over-Retrieval
  3. Post-Filter Vector-Ergebnisse (Tags/Folders/After über JOIN-Metadaten)
  4. `store.bm25_search(query, top_k=top_k * 2, filters=...)` — Pre-Filtered
  5. `rrf_fusion(vector_results, bm25_results, k=60)` → fusionierte Liste
  6. Top `top_k` zurückgeben
- `get_similar(path: str, top_k: int) -> list[SearchResult]`
  1. Alle Chunks der Notiz laden
  2. Durchschnittsvektor berechnen
  3. Vector Search mit Durchschnittsvektor
  4. Eigene Notiz aus Ergebnissen filtern
- `rrf_fusion(results_lists: list[list[SearchResult]], k: int) -> list[SearchResult]` — Reciprocal Rank Fusion

## Block 7: Indexer

### Step 7.1 — Full + Incremental Indexer

**Datei:** `src/mneme/indexer.py`

- `Indexer(store: Store, embedding_provider: EmbeddingProvider, config: MnemeConfig)`
- `index_vault(full: bool = False) -> IndexResult`
  1. Vault scannen: alle `.md` per glob_patterns, exclude_patterns ausschließen
  2. Für jede Datei: `parse_note()` → `content_hash` vergleichen
  3. Wenn `full=True` oder Hash geändert: `chunk_note()` → `embed()` → `store.upsert_note()` + `store.upsert_chunks()`
  4. Wenn Hash gleich: skippen (incremental)
  5. Am Ende: Verwaiste Notes in DB löschen (Pfad existiert nicht mehr im Vault)
  6. Return `IndexResult(indexed, skipped, deleted, duration_seconds)`
- `index_file(path: Path)` — Einzelne Datei indexieren (für File Watcher)
- `remove_file(path: Path)` — Einzelne Datei aus Index entfernen

Batch-Embedding: Chunks sammeln, in Batches à 32 an Provider schicken.

### Step 7.2 — Indexer-Tests

**Datei:** `tests/test_indexer.py`

- Test: index_vault() auf Test-Vault (3-5 .md Dateien in tmp_path) → alle indexiert
- Test: Incremental — zweiter Lauf ohne Änderungen → skipped = N, indexed = 0

## Block 9: File Watcher

### Step 9.1 — Watchdog File Watcher mit Debouncing

**Datei:** `src/mneme/watcher.py`

- `VaultWatcher(vault_path: Path, indexer: Indexer, config: MnemeConfig)`
- Watchdog `FileSystemEventHandler`:
  - `on_created` / `on_modified` → `indexer.index_file(path)`
  - `on_deleted` → `indexer.remove_file(path)`
  - `on_moved` → `remove_file(src)` + `index_file(dest)`
- Filter: nur `.md`, nur innerhalb glob_patterns, nicht in exclude_patterns
- Debouncing: Dict `{path: timestamp}`, Events für gleiche Datei innerhalb 2s ignorieren (Timer-basiert mit `threading.Timer`)
- `start()` / `stop()` Methoden

### Step 9.2 — Watcher-Tests

## Block 10: CLI

### Step 10.1 — Click CLI (setup, serve, reindex, status)

**Datei:** `src/mneme/cli.py`

- `@click.group() main()` — Entry Point
- `@main.command() setup()`:
  1. Vault-Pfad abfragen (click.prompt)
  2. Embedding-Modell bestätigen (Default BGE-M3)
  3. Config-Datei erstellen
  4. Initialer Index (`indexer.index_vault(full=True)`)
  5. Erfolgsmeldung mit Stats
- `@main.command() serve()`:
  1. Config laden
  2. `create_server(config)` → `server.run(transport="stdio")`
- `@main.command() reindex(--full)`:
  1. Config laden, Indexer erstellen
  2. `indexer.index_vault(full=full)`
  3. Stats ausgeben
- `@main.command() status()`:
  1. Config laden, Store öffnen
  2. `store.get_stats()` ausgeben
